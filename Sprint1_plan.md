# Sprint 1 Plan – Course Registration System

**Team Members:** Leonard, Yusuf, Javad, Saleh  
**Date:** March 17, 2026  

---

## 1. Product Vision

The goal of this project is to create a simple and reliable online course registration system that allows students to search for courses, view schedules, and register for classes without confusion. The system will also allow instructors and administrators to manage course offerings and track student enrollment.

The vision is to improve the registration experience by making it faster, more user-friendly, and accessible from any device.

---

## 2. Product Backlog

### User Story 1
As a student, I want to search for courses by keyword so that I can quickly find classes I am interested in.  
**Priority:** High  
**Estimate:** Medium (2–6 hours)  
**Acceptance Criteria:**
- User can enter a keyword  
- System returns matching courses  
- Results display course name, instructor, and time  

---

### User Story 2
As a student, I want to view course details so that I can understand what the course includes.  
**Priority:** High  
**Estimate:** Medium  
**Acceptance Criteria:**
- Displays course description  
- Shows instructor name  
- Shows schedule and credits  

---

### User Story 3
As a student, I want to register for a course so that I can enroll in classes.  
**Priority:** High  
**Estimate:** High (multi-day effort)  
**Acceptance Criteria:**
- User can click “Register”  
- Course is added to schedule  
- System prevents duplicate registration  

---

### User Story 4
As a student, I want to view my schedule so that I can keep track of my classes.  
**Priority:** High  
**Estimate:** Medium  
**Acceptance Criteria:**
- Displays all registered courses  
- Shows times and instructors  

---

### User Story 5
As an admin, I want to add new courses so that the system stays updated.  
**Priority:** Medium  
**Estimate:** Medium  
**Acceptance Criteria:**
- Admin can input course info  
- Course appears in search results  

---

### User Story 6
As an admin, I want to remove courses so that outdated courses are not shown.  
**Priority:** Low  
**Estimate:** Low (<2 hours)  
**Acceptance Criteria:**
- Admin can delete course  
- Course no longer appears  

---

## 4. Sprint Goal

The goal of Sprint 1 is to build the foundation of the system by allowing users to search for courses and view course details. This ensures the core functionality is working before adding more advanced features like registration.

---

## 5. Sprint Backlog

These features are essential because users must be able to find and understand courses before registering. They also form the foundation for all future features.
